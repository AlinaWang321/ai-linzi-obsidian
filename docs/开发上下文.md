# AI霖子 Obsidian 插件开发上下文

> 本文是 Codex、Claude Code 和所有后续开发者的强制开发边界。修改插件或插件 API 前必须完整阅读；安全与产品边界不得以“临时兼容”“减少代码量”或“先上线再说”为由放宽。

## 1. 产品与代码边界：公开插件只是薄客户端

必须始终遵守：

> 公开插件只做界面、本地笔记操作和版本化 API 调用；真正有商业价值的提示词、AI 判断、计费、学员数据和模型编排全部放在私有后端。

公开插件仓库可以包含：

- Obsidian 界面、交互状态和用户主动触发的 Vault 读写。
- 请求参数、响应解析、错误提示与对服务端版本化 API 的调用。
- 不含商业机密的 Markdown 结构解析、公众号内联排版和本地安全校验。
- 可公开审查的隐私、安全、数据边界文档和回归测试。

公开插件仓库不得包含：

- AI霖子的核心系统提示词、商业判断规则、私有方法论提示词和隐藏验收标准。
- DeepSeek、Seedream 等模型的密钥、供应商密钥、真实内部接口或可绕过服务端计费的直连能力。
- 学员资料、长期记忆、账号权益、余额、扣费明细或其他服务端真相源数据。
- 模型选择、降级、重试、计费、风控和多模型编排的核心实现。
- 任何能让第三方仅复制公开仓库就复刻 AI霖子核心能力的代码或配置。

需要新增 AI 能力时，先在私有 AI霖子 webapp 后端实现，再通过 `/api/plugin/v1/*` 暴露最小必要接口。插件不能为省事复制后端提示词或直接连接模型供应商。

## 2. 用户数据与凭证

- 学员 Markdown、配图和内容状态默认留在学员自己的 Vault；插件不得扫描或上传整个 Vault。
- 只有用户明确触发对话、带上当前笔记、喂库、配图、发布或更新笔记时，才读取和发送该动作所需的最小内容。
- Token、公众号 AppSecret 等敏感凭证必须进入 Obsidian `SecretStorage`；不得写入 `data.json`、日志、Git、报错文本或分析事件。
- AI霖子云端负责账号、权益、积分、知识库、插件云端对话及模型调用记录；这些数据不得依赖 Alina 的个人电脑。
- 本地写入必须有明确的用户动作。更新当前笔记、应用局部补丁、覆盖图片与公众号发布都不得静默执行。

## 3. 对话历史：插件到云端的单向可见性

目标关系：

- 插件产生的对话同步到 AI霖子云端，网页版可以查看这些对话。
- 网页版或微信端产生的对话绝不反向同步到 Obsidian 插件。
- 插件“历史”只显示插件产生的对话；插件“清空历史”只删除插件产生的对话。

强制实现规则：

1. 插件会话 `sessionId` 必须使用固定前缀 `obsidian:`。
2. 插件的列表、读取、标题和删除 API 必须在数据库查询层同时限制当前 `student_no` 与 `session_id LIKE 'obsidian:%'`。
3. 禁止依赖客户端传来的 `channel` 作为唯一隔离条件；禁止插件删除接口只按 `student_no` 执行批量删除。
4. 网页端仍可读取学员全部合法云端会话，因此能看到插件会话；插件端只能读取插件命名空间。
5. 旧插件发送普通 UUID 时，新的插件聊天写入 API 在服务端补上 `obsidian:` 前缀。
6. 历史遗留的无前缀云端会话来源不可靠，插件必须排除，不能为了“看起来完整”而冒险显示或删除。旧版本本机 `conversations.json` 本来只保存插件对话，可在客户端读取时安全迁入前缀。
7. 插件清空确认文案必须明确说明：网页版与微信端对话不受影响。

任何修改对话存储、恢复、跨端同步或清空逻辑的提交，都必须同时运行客户端和服务端的会话隔离测试。

## 4. API 与后端边界

- 插件的新能力优先走 `/api/plugin/v1/*`；旧接口只做明确、限时的兼容，不继续扩张。
- 鉴权、授权、套餐校验、积分扣减、幂等、审计和访问范围必须由服务端执行，不能信任客户端 UI 状态。
- API 返回插件完成当前界面动作所需的最小数据，不把内部提示词、供应商响应、模型路由或学员完整档案返回客户端。
- 图片生成走服务端 Seedream 能力；主对话走服务端 AI霖子模型编排。公开插件不得改成直连模型。
- 插件与网页端共享私有后端不等于共享客户端历史视图；每个客户端的读取与删除范围必须由服务端明确限定。

## 5. 每次交付前的最低验收

- `npm run build`
- `npm test`
- 服务端插件会话隔离测试与生产构建通过。
- 检查 Git 变更中没有真实 Token、AppSecret、模型密钥、学员数据、核心提示词或内部模型编排。
- 检查插件删除接口在没有 `sessionId` 时仍保留 `obsidian:` 查询条件。
- 检查网页版对话 API 没有被改成只读插件会话；单向可见性必须保持。
- 把构建后的 `main.js`、`manifest.json`、`styles.css` 安装到测试 Vault，重启 Obsidian 后实测历史列表、恢复和清空。

## 6. 仓库与发布

- 公开仓库：AI霖子 Obsidian 插件薄客户端、公开文档、测试与发布包。
- 私有仓库：AI霖子 webapp、核心提示词、计费、学员数据、模型编排与插件服务端 API。
- 两个仓库独立提交、独立审核和独立发布。修改插件不得顺手改变网页版 UI；修改私有后端必须确认现有网页端接口不回归。
- 发布前先检查本文件；若产品决定改变这些边界，必须由 Alina 明确批准并同步更新本文件、`AGENTS.md` 与 `CLAUDE.md`。
