# AI霖子 Obsidian 插件开发指令

修改本仓库前，必须完整阅读并遵守 [`docs/开发上下文.md`](docs/开发上下文.md)。

不可放宽的核心边界：公开插件只做界面、本地笔记操作和版本化 API 调用；核心提示词、AI 判断、计费、学员数据和模型编排只存在于私有后端。插件会话使用 `obsidian:` 命名空间：插件对话可以同步到云端并被网页版查看，但网页端和微信端对话不得同步回插件，也不得被插件清空。

涉及对话历史、恢复、删除、跨端同步或鉴权的改动，必须同时验证插件端与服务端隔离测试。

勾选当前笔记时，用户在主对话要求给某个 Part/段落新增一张配图，必须走私有后端的当前笔记单图接口，先在对话中预览，用户确认后再插入原文；不得引导回整篇配图技能，也不得用脱离文章的独立生图代替。

通用 AI 生图必须作为主对话中的显式模式提供：支持连续改图，也允许用户关闭「参考上一张图」后开始新图；成功结果自动保存到用户 Vault，并在本地对话中保留图片卡片。云端历史只同步文字，不同步图片数据或本地路径。

公众号配图的专属人偶参考图必须保存在用户 Vault；普通设置只存相对路径，只有用户主动生图时才发送本次所需的压缩图。整篇配图、续跑、对话补图与单图修改必须沿用该设置，参考图丢失则回退通用人偶。

主对话“AI 生图模式”和“我的专属人偶”只对 Pro/Business 开放；通用人偶文章配图保持原有权益。会员权限必须由私有服务端强制复核，公开插件只能做界面提示和提前拦截。

多笔记与文件夹授权只对 Pro/Business 开放。插件只有在用户主动打开选择器时，才可在本地列出 Markdown 笔记路径；只有用户明确勾选并确认的正文才随当前对话发送。授权正文与路径不得写入设置或云端/本机会话历史，新建、切换或结束对话必须清空；禁止自动上传整个 Vault。篇数、单篇字数和合计字数必须由私有服务端再次校验。

“我的工作流”只能保存学员自己编写或从自己对话要求中确认的任务说明，作为 Markdown 留在用户 Vault；不得把 AI霖子的私有提示词、判断规则或模型编排写入工作流文件。工作流只有在用户明确点击“开始运行”后，才可通过 `/api/plugin/v1/chat` 发送任务说明与本轮已授权内容；创建、浏览、编辑和删除工作流都不得调用模型或消耗积分。
